# WPKirk-ReactJS-Boilerplate → WPKirk-NuxtJS

Vue admin app built with **Nuxt 3 + Vue 3 Composition API** — no third-party UI
kit. Everything uses WordPress admin CSS classes and the `wp.i18n.*` globals, so
the bundle stays small and the look matches native wp-admin.

## What this demos

`pages/index.vue` renders a small admin page wired through the Vue 3 Composition
API and Nuxt 3:

- **State** — `ref()` from Vue for reactive values
- **Composable** — `composables/use-counter.ts` showcases a typed reusable composable
- **i18n** — `composables/useWpI18n.ts` wraps `window.wp.i18n.__()`, `_n()`, `sprintf()`
- **Conditional rendering** — `v-if` directive
- **List rendering** — `v-for` directive
- **Reactivity** — `v-model` for two-way data binding
- **File-based routing** — `pages/` directory auto-registered by Nuxt
- **Head management** — `useHead()` composable

**Key files to read first:**

| File | What to look at |
| --- | --- |
| `pages/index.vue` | Full demo composition using Vue 3 Composition API |
| `composables/use-counter.ts` | Typed custom composable pattern |
| `composables/__tests__/use-counter.test.ts` | Vitest over the composable |

## Smoke test (manual, ~30s)

With the plugin active:

1. Log in to `wp-admin` and open **WP Kirk React → Main View**.
2. Verify the demo renders: a counter with increment/decrement/reset buttons, a
   text input with live v-model greeting, a toggle section, and a list renderer.
3. Type into the text input — the greeting below it should update live (proves `v-model`).
4. Click **Increment** / **Decrement** — count goes up/down; **Reset** returns to 0.
5. Click **Show message** / **Hide message** — conditional block toggles (proves `v-if`).
6. Click **Add item** — a new list item appears (proves `v-for` reactivity).
7. Run `yarn test` locally — composable tests should pass.

If any of the above fail: check devtools Console for Vue errors, `wp-content/debug.log`
for PHP errors, and `yarn build` output for missing dependencies.

## Use as a template

```sh
# 1. clone from the GitHub template
gh repo create my-vue-plugin --template wpbones/WPKirk-ReactJS-Boilerplate --public --clone
cd my-vue-plugin

# 2. rename the PHP namespace + plugin slug
composer install
php bones rename "My Vue Plugin"

# 3. build + activate
yarn install && yarn build
wp plugin activate my-vue-plugin
```

Each `.vue` file in `pages/` becomes a Nuxt route (and a PHP-enqueueable app bundle).
The Nuxt client bundle is post-processed into `public/apps/app.js` by `scripts/post-build.mjs`.

## Framework surface exercised

This boilerplate is the **regression bed for the Nuxt + WordPress Vue runtime path**:

- `View::withAdminAppsScript()` + `nuxi build` → `public/apps/app.js` (concatenated)
- Nuxt SPA mode (`ssr: false`) — client-only build, no server needed
- `scripts/post-build.mjs` concatenates Nuxt output chunks into a single deployable JS file
- `public/apps/app.asset.php` dependencies (auto-generated by post-build script)
- `wp_set_script_translations()` wiring via WordPress global `wp.i18n`
- Vitest with `@vue/test-utils` and `jsdom` for composable and component tests
