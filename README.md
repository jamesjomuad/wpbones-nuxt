# WP Bones + Nuxt 3 Admin SPA

A WordPress plugin skeleton that replaces the traditional React/Gatsby admin UI with a **Nuxt 3 SPA** rendered inside a **Shadow DOM** for full CSS isolation from the WordPress admin. Uses **@nuxt/ui v2** for components and integrates with the WP Bones PHP framework via a bootloader-based asset pipeline.

## Features

- **Nuxt 3** SPA (`ssr: false`) with hash-mode routing (`#/`, `#/tools`, `#/components`)
- **Shadow DOM** — the entire Vue app mounts inside `#__nuxt`'s shadow root, isolating Tailwind/Nuxt UI styles from WP admin CSS
- **@nuxt/ui v2** — 30+ auto-imported components (Button, Card, Modal, Table, Tabs, Accordion, etc.)
- **WP Bones integration** — PHP views, menu registration, asset enqueuing via `AdminAppsAssetEnqueuer`
- **Bootloader** — auto-detects its own script URL, sets `window.__NUXT__`, loads module JS/CSS from `public/build/`
- **TypeScript**, **Vitest**, **Prettier**
- **Hash mode** avoids URL conflicts with WordPress admin paths

## Stack

| Layer | Technology |
|-------|-----------|
| SPA | Nuxt 3 + Vue 3 + Vite |
| UI | @nuxt/ui v2 (Tailwind CSS v3) |
| Isolation | Shadow DOM |
| Backend | WP Bones PHP framework |
| Routing | Vue Router hash mode |
| i18n | `wp.i18n` globals |
| Testing | Vitest |

## Project Structure

```
resources/js/              ← Nuxt source files
├── pages/                 ← Route pages (index, tools, components)
├── composables/           ← Vue composables (use-counter, useWpI18n)
├── plugins/               ← Nuxt plugins (shadow-dom.client.ts)
├── app.vue                ← App shell
├── app/router.options.ts  ← Hash mode config
└── assets/css/main.css    ← Custom styles

public/
├── build/                 ← Built JS/CSS (generated by yarn build)
├── apps/                  ← Bootloader + asset.php (generated)
├── js/                    ← Vendor JS (prism)
└── css/                   ← Vendor CSS (prism)

plugin/
├── Http/Controllers/      ← DashboardController (index, tools, components)
└── ...

resources/views/dashboard/ ← PHP views
config/menus.php           ← Admin menu items
```

## Requirements

- PHP 8.0+
- WordPress 6.0+
- Node.js 18+
- Yarn

## Installation

```bash
cd /wp-content/plugins/
git clone <repo-url> wpbones-nuxt
cd wpbones-nuxt
composer install
yarn install
yarn build
```

Activate the plugin from the WordPress admin plugins page. A **NuxtJS** menu item will appear in the admin sidebar.

## Development

For local development, run `yarn dev` alongside your WordPress instance. Navigate to any plugin admin page — the bootloader loads the latest build from the dev server.

### Adding a New Page

1. Create `resources/js/pages/my-page.vue`
2. Create `resources/views/dashboard/my-page.php` with `<div id="__nuxt" data-initial-route="/my-page">`
3. Add a controller method in `plugin/Http/Controllers/Dashboard/DashboardController.php`
4. Register the menu item in `config/menus.php`

### Using Nuxt UI Components

All `@nuxt/ui` v2 components are auto-imported with the `U` prefix. No manual imports needed:

```vue
<UButton label="Click me" color="primary" />
<UCard>Content</UCard>
<UModal v-model="open">...</UModal>
```

Icons use the `heroicons` collection by default:

```vue
<UIcon name="heroicons:cog-6-tooth" />
<UButton icon="heroicons:plus" label="Add" />
```
